## 3.2 表结构设计

### notes 表（笔记表）

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY | 笔记 ID，自增主键 |
| content | TEXT | NOT NULL | 笔记内容（Markdown） |
| files | TEXT | DEFAULT '[]' | 文件附件列表（JSON 数组） |
| created_at | INTEGER | NOT NULL | 创建时间戳（毫秒） |
| updated_at | INTEGER | NOT NULL | 更新时间戳（毫秒） |
| is_pinned | BOOLEAN | DEFAULT 0 | 是否置顶 |
| is_favorited | INTEGER | DEFAULT 0 NOT NULL | 是否收藏 |
| is_archived | INTEGER | DEFAULT 0 NOT NULL | 是否归档 |
| pics | TEXT | | 图片 URL 列表（JSON 数组） |
| videos | TEXT | DEFAULT '[]' | 视频 URL 列表（JSON 数组） |

**来源**：[文件路径：../memos-worker/src/schema.sql](../memos-worker/src/schema.sql) 第 6-17 行

### tags 表（标签表）

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 标签 ID，自增主键 |
| name | TEXT | NOT NULL UNIQUE | 标签名称（唯一） |

**来源**：[文件路径：../memos-worker/src/schema.sql](../memos-worker/src/schema.sql) 第 1-4 行

### note_tags 表（笔记标签关联表）

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| note_id | INTEGER | NOT NULL, FK | 笔记 ID，外键关联 notes.id |
| tag_id | INTEGER | NOT NULL, FK | 标签 ID，外键关联 tags.id |

**主键**：(note_id, tag_id) 复合主键

**外键约束**：
- note_id → notes.id (ON DELETE CASCADE)
- tag_id → tags.id (ON DELETE CASCADE)

**来源**：[文件路径：../memos-worker/src/schema.sql](../memos-worker/src/schema.sql) 第 19-25 行

### nodes 表（知识库节点表）

| 字段名 | 类型 | 约束 | 说明 |
|-------|------|------|------|
| id | TEXT | PRIMARY KEY | 节点 ID（UUID） |
| type | TEXT | NOT NULL | 节点类型 |
| title | TEXT | NOT NULL | 节点标题 |
| content | TEXT | | 节点内容 |
| parent_id | TEXT | FK | 父节点 ID，外键关联 nodes.id |
| created_at | INTEGER | NOT NULL | 创建时间戳 |
| updated_at | INTEGER | NOT NULL | 更新时间戳 |

**来源**：[文件路径：../memos-worker/src/schema.sql](../memos-worker/src/schema.sql) 第 28-37 行

## 3.3 全文搜索设计

系统使用 SQLite FTS5 虚拟表实现全文搜索功能。

### notes_fts 虚拟表

| 字段 | 说明 |
|------|------|
| content | 笔记内容 |
| files | 文件列表（JSON） |

**配置**：
- content = 'notes'（关联基础表）
- content_rowid = 'id'（关联 rowid）

**来源**：[文件路径：../memos-worker/src/schema.sql](../memos-worker/src/schema.sql) 第 43-48 行

### 同步触发器

为了保持 FTS 索引与基础表同步，定义了三个触发器：

1. **notes_after_insert**：插入笔记后自动添加到 FTS 索引
2. **notes_after_delete**：删除笔记后自动从 FTS 索引删除
3. **notes_after_update**：更新笔记后自动更新 FTS 索引

**来源**：[文件路径：../memos-worker/src/schema.sql](../memos-worker/src/schema.sql) 第 55-66 行

## 3.4 索引设计

除了主键和唯一约束自动创建的索引外，系统未显式创建额外索引。对于当前规模的个人应用，现有索引已足够满足查询性能需求。

## 3.5 KV 存储设计

除了关系型数据库，系统还使用 Cloudflare KV 存储以下数据：

| Key 前缀 | 用途 | 说明 |
|---------|------|------|
| `session:` | 会话存储 | 存储用户会话信息，TTL 30 天 |
| `user_settings` | 用户设置 | 存储用户个性化配置 |
